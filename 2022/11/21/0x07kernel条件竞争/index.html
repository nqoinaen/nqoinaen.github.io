0x07 kernel 条件竞争

本文最后更新于：2022年11月21日 晚上

前言

本篇是关于kernel pwn中的条件竞争，由于是我们在用户态编写程序与内核交互，所以我们可以启动多个线程同时运行，以达到条件竞争

double fetch

double fetch可以理解为两次或多次对同一个对象取值

例：内核对一个对象第一次取值进行合法性检测，第二次取值进行使用

我们可以在第一次取值后对值进行修改成恶意数据，在第二次使用时执行我们的恶意数据

通过在 first fetch 与 second fetch 之间的空挡修改数据从而改变内核执行流的利用手法便被称之为double fetch

第一个检查判断我们输入的结构体是否在用户空间

第二个检查判断我们输入的flag_str是否在用户空间

第三个检查判断我们输入的flag_str的len是否等于题目给的flag的长度

所有检查通过后对逐个字符与题目给的flag进行比较，都比较成功了的话打印题目的flag

0x02解法1：条件竞争

正常来讲我们用户态是无法使用内核态的指针

首先0x6666选项得到flag的地址

接着创建竞争线程，一直重复修改我们与内核交互的.flag_addr指针为内核flag的地址

原进程则一直通过0x1337选项与内核交互，由于条件竞争，检查通过原本buf在用户态的指针，再次使用该指针时，已经由于我们条件竞争指向了内核态的flag地址，程序就告诉了我们flag

0x03解法2：测信道攻击

由于是 flag 是硬编码的，并且是检查方法是逐字节比较，因此可以逐字节爆破来得到 flag。

方法是将待爆破的字节放在 mmap 申请的内存页末位，此时下一字节位于不可读写的用户态空间。

如果我们成功的猜对了一个字节，就会继续往下读，下一字节位于不可读写的用户态空间，读取会导致kernel panic，从而可以判断是否爆破的一个字节正确。
